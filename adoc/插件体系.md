## 1. 插件体系概览

OpenClaw 的核心设计原则是**核心保持插件无关**——所有具体能力（LLM 提供者、消息渠道、工具、记忆后端、语音、图像生成等）都通过插件实现。核心只提供运行时骨架和 Plugin SDK 接口。

```
核心（src/）
  ↕ 只通过 openclaw/plugin-sdk/* 和 manifest 元数据交互
插件（extensions/ 或第三方 npm 包）
  ↕ 不能访问核心内部实现（src/**、src/plugin-sdk-internal/**）
```

### 插件的两种形态

| 形态 | 说明 | 示例 |
|------|------|------|
| **捆绑插件（bundled）** | 随 OpenClaw 核心发布，在 `extensions/` 目录下 | `extensions/openai/`、`extensions/telegram/` |
| **外部插件（external）** | 独立 npm 包，通过 `pnpm openclaw plugins install` 安装 | 第三方社区插件 |

捆绑插件和外部插件使用完全相同的 SDK 接口，区别只在于分发方式。

## 2. 插件加载流程

### 从 Pi Runner 到插件注册

Pi Runner 在每次 Agent 执行前调用 `ensureRuntimePluginsLoaded`，确保插件已加载：

```
src/agents/pi-embedded-runner/run.ts:536
  ensureRuntimePluginsLoaded({
    config,
    workspaceDir,
    allowGatewaySubagentBinding,
  })
    ↓
src/agents/runtime-plugins.ts
  ensureStandaloneRuntimePluginRegistryLoaded({
    requiredPluginIds,   // 从当前 metadata snapshot 读取
    loadOptions: { config, workspaceDir, onlyPluginIds }
  })
    ↓
src/plugins/runtime/standalone-runtime-registry-loader.ts
  加载插件注册表（带缓存，进程内复用）
    ↓
src/plugins/loader.ts
  逐个加载插件：
    1. 读取 openclaw.plugin.json（manifest）
    2. 动态 import 插件入口模块
    3. 调用 plugin.register(api)
    4. 将注册结果写入 PluginRegistry
    ↓
src/plugins/runtime.ts
  setActivePluginRegistry(registry)
  全局单例，进程内所有 Agent 共享
```

### 缓存策略

插件注册表是**进程稳定**的——manifest、catalog、生成路径、捆绑元数据在进程生命周期内不变。热路径不会重新扫描或重新加载插件。配置变更需要重启或显式 `openclaw doctor --fix`。

## 3. 插件注册表（PluginRegistry）

`src/plugins/registry-types.ts` 定义了 `PluginRegistry`，它是所有已加载插件贡献的能力的聚合视图：

```typescript
type PluginRegistry = {
  plugins: PluginRecord[]                    // 所有插件的元数据记录
  tools: PluginToolRegistration[]            // 注册的工具
  hooks: PluginHookRegistration[]            // 注册的 Hook
  channels: PluginChannelRegistration[]      // 注册的渠道
  providers: PluginProviderRegistration[]    // 注册的 LLM 提供者
  webSearchProviders: ...[]                  // 网络搜索提供者
  webFetchProviders: ...[]                   // 网络抓取提供者
  speechProviders: ...[]                     // 语音提供者
  imageGenerationProviders: ...[]            // 图像生成提供者
  videoGenerationProviders: ...[]            // 视频生成提供者
  musicGenerationProviders: ...[]            // 音乐生成提供者
  embeddingProviders: ...[]                  // 嵌入向量提供者
  memoryEmbeddingProviders: ...[]            // 记忆嵌入提供者
  services: PluginServiceRegistration[]      // 后台服务
  httpRoutes: PluginHttpRouteRegistration[]  // HTTP 路由
  commands: PluginCommandRegistration[]      // CLI 命令
  sessionExtensions: ...[]                   // Session 扩展
  runtimeLifecycles: ...[]                   // 运行时生命周期钩子
  agentEventSubscriptions: ...[]             // Agent 事件订阅
  // ... 共 40+ 个注册槽
}
```

注册表有三个"视图"，分别服务不同场景：

| 视图 | 访问函数 | 用途 |
|------|---------|------|
| 活跃注册表 | `getActivePluginRegistry()` | Agent 执行时的主注册表 |
| HTTP 路由注册表 | `getActivePluginHttpRouteRegistry()` | 处理 HTTP 请求时使用 |
| 渠道注册表 | `getActivePluginChannelRegistry()` | 渠道消息分发时使用 |

## 4. 插件 Manifest（openclaw.plugin.json）

每个插件根目录下必须有 `openclaw.plugin.json`，这是插件的**静态元数据**，在不执行插件代码的情况下就能被发现和规划。

### 完整字段说明

```jsonc
{
  // 必填：全局唯一插件 ID，小写字母+连字符
  "id": "my-plugin",

  // 激活策略：控制何时加载插件代码
  "activation": {
    "onStartup": false,          // true = 网关启动时立即加载
    "onProviders": ["openai"],   // 当这些 provider 被配置时加载
    "onChannels": ["telegram"],  // 当这些渠道被启用时加载
    "onCommands": ["my-cmd"],    // 当这些命令被调用时加载
    "onConfigPaths": ["tools.web.search.apiKey"]  // 当配置中有这些路径时加载
  },

  // 插件类型（可选，通常由 register() 内容决定）
  "kind": "provider",  // "provider" | "channel" | "tool" | "service" | ...

  // 声明此插件提供的能力契约（用于激活规划，不执行代码）
  "contracts": {
    "webSearchProviders": ["my-search"],
    "providers": ["my-llm"]
  },

  // 配置 Schema（JSON Schema 格式）
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": { "type": "string" }
    }
  },

  // 配置项的 UI 提示（Control UI 中显示）
  "uiHints": {
    "apiKey": {
      "label": "API Key",
      "help": "Your API key",
      "sensitive": true,
      "placeholder": "sk-..."
    }
  },

  // 提供者认证相关的环境变量（用于无代码发现）
  "providerAuthEnvVars": {
    "my-provider": ["MY_API_KEY"]
  },

  // 设置/引导流程元数据
  "setup": {
    "providers": [
      {
        "id": "my-provider",
        "authMethods": ["api-key"],
        "envVars": ["MY_API_KEY"]
      }
    ]
  }
}
```

### 激活策略详解

`activation.onStartup: false` 是推荐默认值——插件按需加载，减少启动时间。只有需要在网关启动时就注册渠道监听或后台服务的插件才设为 `true`。

## 5. 插件入口（definePluginEntry）

插件的 JS 入口文件（`index.ts`）默认导出一个通过 `definePluginEntry` 创建的对象：

```typescript
// index.ts
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";

export default definePluginEntry({
  id: "my-plugin",          // 必须与 manifest 中的 id 一致
  name: "My Plugin",        // 显示名称
  description: "...",       // 描述

  // 可选：配置 Schema（Zod 或自定义 validate）
  configSchema: buildPluginConfigSchema(MyConfigSchema),

  // 核心：注册函数，接收 api 对象，调用各种 register* 方法
  register(api) {
    api.registerWebSearchProvider(createMySearchProvider());
    api.registerTool("my-tool", createMyTool());
    // ...
  },
});
```

`register(api)` 是同步函数，在插件加载时执行一次。所有注册操作都通过 `api` 对象完成。

## 6. OpenClawPluginApi —— 所有注册方法

`OpenClawPluginApi` 是插件与核心交互的唯一接口，包含 50+ 个注册方法，按能力域分组：

### 工具与 Hook

```typescript
api.registerTool(name, factory, options?)
  // 注册一个 Agent 可调用的工具

api.registerHook(name, handler, options?)
  // 注册生命周期 Hook（before_agent_reply、after_tool_call 等）
```

### LLM 提供者

```typescript
api.registerProvider(provider)
  // 注册 LLM 提供者（OpenAI 兼容、Anthropic、自定义等）

api.registerModelCatalogProvider(provider)
  // 注册模型目录提供者（提供可用模型列表）
```

### 搜索与网络

```typescript
api.registerWebSearchProvider(provider)
  // 注册网络搜索提供者（DuckDuckGo、Brave、Exa 等）

api.registerWebFetchProvider(provider)
  // 注册网页抓取提供者
```

### 媒体生成

```typescript
api.registerImageGenerationProvider(provider)
api.registerVideoGenerationProvider(provider)
api.registerMusicGenerationProvider(provider)
```

### 语音与实时

```typescript
api.registerSpeechProvider(provider)           // TTS
api.registerRealtimeTranscriptionProvider(p)   // 实时语音转文字
api.registerRealtimeVoiceProvider(provider)    // 实时语音对话
```

### 渠道

```typescript
api.registerChannel(channelPlugin)
  // 注册消息渠道（Telegram、Slack 等）
```

### 记忆

```typescript
api.registerMemoryCapability(capability)
api.registerMemoryRuntime(runtime)
api.registerMemoryEmbeddingProvider(provider)
api.registerMemoryPromptSection(section)
api.registerMemoryFlushPlan(plan)
```

### 服务与 HTTP

```typescript
api.registerService(service)
  // 注册后台服务（随网关启动/停止）

api.registerHttpRoute(path, handler, options)
  // 注册 HTTP 路由（插件自定义 API 端点）
```

### Session 与 Agent 事件

```typescript
api.registerSessionExtension(extension)
  // 注册 Session 扩展（向 Session 注入额外数据）

api.registerAgentEventSubscription(subscription)
  // 订阅 Agent 事件（turn_start、tool_call 等）

api.emitAgentEvent(params)
  // 向其他插件发送 Agent 事件

api.registerRuntimeLifecycle(lifecycle)
  // 注册运行时生命周期钩子（onStart、onStop）
```

### 上下文引擎

```typescript
api.registerContextEngine(engineId, factory)
  // 注册自定义上下文引擎（替换默认的消息组装逻辑）

api.registerCompactionProvider(provider)
  // 注册自定义上下文压缩提供者
```

### CLI 与命令

```typescript
api.registerCli(registrar, options)
  // 注册 CLI 子命令

api.registerCommand(command)
  // 注册 Agent 可调用的命令
```

### 嵌入向量

```typescript
api.registerEmbeddingProvider(provider)
  // 注册嵌入向量提供者（用于语义搜索）
```

## 7. 三类典型插件详解

### 7.1 网络搜索插件（最简单）

以 DuckDuckGo 插件为例，这是代码量最少的插件类型之一，无需 API Key。

**目录结构：**
```
extensions/duckduckgo/
├── openclaw.plugin.json   # manifest
├── index.ts               # 插件入口
├── package.json
└── src/
    ├── ddg-search-provider.ts   # 搜索提供者实现
    └── ddg-client.ts            # HTTP 客户端
```

**manifest（`openclaw.plugin.json`）：**
```json
{
  "id": "duckduckgo",
  "activation": { "onStartup": false },
  "contracts": {
    "webSearchProviders": ["duckduckgo"]
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "webSearch": {
        "type": "object",
        "properties": {
          "region": { "type": "string" },
          "safeSearch": { "type": "string", "enum": ["strict", "moderate", "off"] }
        }
      }
    }
  }
}
```

**入口（`index.ts`）：**
```typescript
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { createDuckDuckGoWebSearchProvider } from "./src/ddg-search-provider.js";

export default definePluginEntry({
  id: "duckduckgo",
  name: "DuckDuckGo Plugin",
  description: "Bundled DuckDuckGo web search plugin",
  register(api) {
    api.registerWebSearchProvider(createDuckDuckGoWebSearchProvider());
  },
});
```

**搜索提供者实现（`src/ddg-search-provider.ts`）：**
```typescript
import { readNumberParam, readStringParam } from "openclaw/plugin-sdk/param-readers";
import type { WebSearchProviderPlugin } from "openclaw/plugin-sdk/provider-web-search-contract";

export function createDuckDuckGoWebSearchProvider(): WebSearchProviderPlugin {
  return {
    id: "duckduckgo",
    // 工具 Schema 定义
    createTool: (ctx) => ({
      description: "Search the web using DuckDuckGo.",
      parameters: {
        type: "object",
        properties: {
          query: { type: "string", description: "Search query string." },
          count: { type: "number", minimum: 1, maximum: 10 },
        },
      },
      // 工具执行逻辑
      execute: async (args) => {
        const query = readStringParam(args, "query", { required: true });
        // ... 调用 DuckDuckGo API
        return results;
      },
    }),
  };
}
```

### 7.2 LLM 提供者插件

以 OpenAI 插件为例，这是最复杂的插件类型，需要处理认证、模型目录、流式传输等。

**manifest 关键字段：**
```json
{
  "id": "openai",
  "activation": { "onStartup": false, "onProviders": ["openai"] },
  "kind": "provider",
  "modelSupport": {
    "modelPrefixes": ["gpt-", "o1", "o3", "o4"]
  },
  "providerAuthEnvVars": {
    "openai": ["OPENAI_API_KEY"]
  },
  "setup": {
    "providers": [{ "id": "openai", "authMethods": ["api-key"], "envVars": ["OPENAI_API_KEY"] }]
  }
}
```

**入口（`index.ts`）：**
```typescript
export default definePluginEntry({
  id: "openai",
  name: "OpenAI Plugin",
  description: "OpenAI provider plugin",
  register(api) {
    api.registerProvider(createOpenAIProvider());
    api.registerModelCatalogProvider(createOpenAIModelCatalog());
  },
});
```

**提供者实现核心接口（`ProviderPlugin`）：**
```typescript
type ProviderPlugin = {
  id: string
  // 认证方法列表
  authMethods?: ProviderAuthMethod[]
  // 模型目录（静态）
  catalog?: ProviderPluginCatalog
  // 流式请求处理
  createStream: (ctx: ProviderStreamContext) => AsyncIterable<StreamChunk>
  // 模型 ID 规范化
  normalizeModelId?: (ctx: ProviderNormalizeModelIdContext) => string | undefined
  // 运行时认证准备
  prepareRuntimeAuth?: (ctx: ProviderPrepareRuntimeAuthContext) => ProviderPreparedRuntimeAuth
}
```

### 7.3 渠道插件

渠道插件使用 `defineChannelPluginEntry`（而非 `definePluginEntry`），需要实现 `ChannelPlugin` 接口。

**入口（`index.ts`）：**
```typescript
import { defineChannelPluginEntry } from "openclaw/plugin-sdk/core";

export default defineChannelPluginEntry({
  id: "my-channel",
  name: "My Channel Plugin",
  description: "...",
  register(api) {
    api.registerChannel(createMyChannelPlugin());
  },
});
```

**渠道插件核心接口（`ChannelPlugin`）：**
```typescript
type ChannelPlugin = {
  id: ChannelId
  // 启动监听（返回停止函数）
  startListening: (ctx: ChannelListenContext) => Promise<() => void>
  // 发送消息
  sendMessage: (ctx: ChannelSendContext) => Promise<ChannelSendResult>
  // 获取账号信息
  getAccountInfo?: () => Promise<ChannelAccountInfo>
  // 渠道配置 Schema
  configSchema?: ChannelConfigSchema
}
```

## 8. 从零制作一个插件

下面以制作一个**自定义网络搜索插件**（调用 Tavily API）为例，演示完整流程。

### 第一步：创建目录结构

```
my-tavily-plugin/
├── openclaw.plugin.json
├── index.ts
├── package.json
└── src/
    └── tavily-search-provider.ts
```

### 第二步：编写 manifest

```json
// openclaw.plugin.json
{
  "id": "my-tavily",
  "activation": {
    "onStartup": false
  },
  "providerAuthEnvVars": {
    "my-tavily": ["TAVILY_API_KEY"]
  },
  "uiHints": {
    "webSearch.apiKey": {
      "label": "Tavily API Key",
      "help": "Your Tavily API key. Get one at https://tavily.com",
      "sensitive": true,
      "placeholder": "tvly-..."
    }
  },
  "contracts": {
    "webSearchProviders": ["my-tavily"]
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "webSearch": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "apiKey": { "type": ["string", "object"] },
          "maxResults": { "type": "number" }
        }
      }
    }
  }
}
```

### 第三步：编写搜索提供者

```typescript
// src/tavily-search-provider.ts
import { readNumberParam, readStringParam } from "openclaw/plugin-sdk/param-readers";
import type { WebSearchProviderPlugin } from "openclaw/plugin-sdk/provider-web-search-contract";

export function createTavilySearchProvider(): WebSearchProviderPlugin {
  return {
    id: "my-tavily",

    createTool: (ctx) => ({
      description:
        "Search the web using Tavily AI Search. Returns high-quality, AI-curated results.",
      parameters: {
        type: "object",
        required: ["query"],
        additionalProperties: false,
        properties: {
          query: {
            type: "string",
            description: "The search query.",
          },
          maxResults: {
            type: "number",
            description: "Maximum number of results (1-10).",
            minimum: 1,
            maximum: 10,
          },
        },
      },

      execute: async (args) => {
        const query = readStringParam(args, "query", { required: true });
        const maxResults = readNumberParam(args, "maxResults", { integer: true }) ?? 5;

        // 从插件配置读取 API Key
        const apiKey =
          (ctx.config as Record<string, unknown>)?.webSearch?.apiKey ??
          process.env.TAVILY_API_KEY;

        if (!apiKey || typeof apiKey !== "string") {
          return { error: "Tavily API key not configured." };
        }

        const response = await fetch("https://api.tavily.com/search", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${apiKey}`,
          },
          body: JSON.stringify({ query, max_results: maxResults }),
        });

        if (!response.ok) {
          return { error: `Tavily API error: ${response.status}` };
        }

        const data = await response.json();
        return {
          results: data.results.map((r: Record<string, string>) => ({
            title: r.title,
            url: r.url,
            snippet: r.content,
          })),
        };
      },
    }),
  };
}
```

### 第四步：编写插件入口

```typescript
// index.ts
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { createTavilySearchProvider } from "./src/tavily-search-provider.js";

export default definePluginEntry({
  id: "my-tavily",
  name: "Tavily Search Plugin",
  description: "Web search powered by Tavily AI",
  register(api) {
    api.registerWebSearchProvider(createTavilySearchProvider());
  },
});
```

### 第五步：编写 package.json

```json
{
  "name": "openclaw-plugin-my-tavily",
  "version": "1.0.0",
  "description": "Tavily web search plugin for OpenClaw",
  "type": "module",
  "main": "./index.ts",
  "peerDependencies": {
    "openclaw": "*"
  },
  "openclaw": {
    "extensions": ["./index.ts"]
  }
}
```

## 9. 插件安装方式

### 方式一：从 npm 安装（推荐）

```bash
# 安装已发布到 npm 的插件
pnpm openclaw plugins install openclaw-plugin-my-tavily

# 安装特定版本
pnpm openclaw plugins install openclaw-plugin-my-tavily@1.2.0

# 安装后查看状态
pnpm openclaw plugins status
```

### 方式二：从本地路径安装

```bash
# 安装本地开发中的插件
pnpm openclaw plugins install /path/to/my-tavily-plugin

# 或者使用相对路径
pnpm openclaw plugins install ./my-tavily-plugin
```

### 方式三：从 Git 仓库安装

```bash
pnpm openclaw plugins install github:myuser/openclaw-plugin-my-tavily
```

### 方式四：捆绑插件（仅限核心开发）

将插件放入 `extensions/` 目录，并在 `src/plugins/bundled-sources.ts` 中注册。这是核心团队维护的官方插件路径，第三方插件不走这条路。

### 安装后配置

安装完成后，在 `~/.openclaw/openclaw.json` 中配置插件：

```json
{
  "plugins": {
    "entries": {
      "my-tavily": {
        "enabled": true,
        "config": {
          "webSearch": {
            "apiKey": "tvly-your-api-key-here"
          }
        }
      }
    }
  }
}
```

或者通过 CLI 配置：

```bash
# 交互式配置
pnpm openclaw plugins configure my-tavily

# 直接设置配置项
pnpm openclaw config set plugins.entries.my-tavily.config.webSearch.apiKey "tvly-..."
```

### 验证安装

```bash
# 查看所有插件状态
pnpm openclaw plugins status

# 查看特定插件详情
pnpm openclaw plugins status my-tavily

# 运行健康检查
pnpm openclaw doctor
```

## 10. 插件边界规则

这些规则来自 `extensions/AGENTS.md`，是插件开发的硬性约束：

### 导入规则

```typescript
// ✅ 正确：只从 plugin-sdk 导入
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import type { WebSearchProviderPlugin } from "openclaw/plugin-sdk/provider-web-search-contract";
import { readStringParam } from "openclaw/plugin-sdk/param-readers";

// ❌ 错误：不能导入核心内部实现
import { something } from "../../src/agents/agent-command.js";  // 禁止
import { other } from "../../src/plugin-sdk-internal/...";      // 禁止

// ❌ 错误：不能跨插件导入
import { helper } from "../other-plugin/src/helper.js";         // 禁止
```

### 依赖规则

- 插件的运行时依赖属于插件自己的 `package.json`，不能移到根目录
- 如果依赖有运行时 peer，在插件的 `package.json` 中声明/提供
- 使用精确版本（`"1.2.3"`），不用开放范围（`"^1.2.3"`）

### 元数据规则

- `openclaw.plugin.json` 中的元数据必须准确，发现和设置流程依赖它
- 不依赖 import-time 副作用来让插件"可用"——可用性来自 manifest 所有权 + 激活规划
- 控制平面元数据（发现、配置验证、设置提示）尽量用 manifest/descriptor 表达，不要执行运行时代码

### 架构规则

- 提供者的认证、引导、目录选择、厂商特定行为保持在插件本地，不要移入核心
- 如果两个插件共享相同的逻辑（replay policy、tool schema compat 等），提取共享 helper，不要复制
- 不要用 `@ts-nocheck`；lint 抑制注释需要解释原因

## 附录：Plugin SDK 主要入口点

| 入口路径 | 用途 |
|---------|------|
| `openclaw/plugin-sdk/plugin-entry` | `definePluginEntry`，非渠道插件的入口 |
| `openclaw/plugin-sdk/core` | `defineChannelPluginEntry`，渠道插件的入口 |
| `openclaw/plugin-sdk/provider-entry` | 提供者插件的辅助工具 |
| `openclaw/plugin-sdk/param-readers` | 工具参数读取工具（`readStringParam` 等） |
| `openclaw/plugin-sdk/provider-web-search-contract` | `WebSearchProviderPlugin` 类型 |
| `openclaw/plugin-sdk/channel-contract` | `ChannelPlugin` 类型 |
| `openclaw/plugin-sdk/provider-stream-shared` | 流式传输工具 |
| `openclaw/plugin-sdk/memory-core` | 记忆能力接口 |
| `openclaw/plugin-sdk/runtime` | 运行时工具（config、logger 等） |

完整入口点列表见 `scripts/lib/plugin-sdk-entrypoints.json`。
