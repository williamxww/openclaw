## 1. 环境要求

| 工具    | 最低版本                    | 说明                            |
| ------- | --------------------------- | ------------------------------- |
| Node.js | **22.19.0**（推荐 Node 24） | `package.json` engines 字段要求 |
| pnpm    | **11.2.2**                  | `packageManager` 字段锁定版本   |
| WSL2    | Ubuntu 22.04+               | Windows 下推荐在 WSL2 中操作    |
| VS Code | 最新版                      | 含 WSL 扩展，用于断点调试       |

> **为什么用 WSL2？**  
> 构建脚本（`scripts/build-all.mjs`）依赖 Unix 工具链，Windows 原生 CMD/PowerShell 下部分脚本会失败。在 WSL2 中操作最稳定。

## 2. WSL2 环境准备

### 2.1 进入 WSL 并切换到项目目录

```bash
# 在 Windows Terminal 中打开 WSL
wsl

# 项目在 Windows 的 D: 盘，WSL 下路径为：
cd /mnt/d/repository/github/openclaw
```

### 2.2 安装 Node.js（推荐用 nvm）

```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc

# 安装 Node 24（满足 >=22.19.0 要求，并与本仓库 VS Code 调试配置一致）
nvm install 24.11.0
nvm use 24.11.0
node --version   # 应输出 v24.11.0
```

### 2.3 安装 pnpm

```bash
# 通过 corepack 安装（Node 24 自带）
corepack enable
corepack prepare pnpm@11.2.2 --activate

pnpm --version   # 应输出 11.2.2
```

## 3. 安装依赖

```bash
cd /mnt/d/repository/github/openclaw

# 安装所有依赖（含 workspace 子包）
pnpm install
```

安装完成后目录结构中会出现：

- `node_modules/` — 根依赖
- `packages/*/node_modules/` — 子包依赖
- `extensions/*/node_modules/` — 插件依赖

> **注意：** 首次安装可能需要几分钟，pnpm 会并发安装所有 workspace 包。

## 4. 构建项目

### 4.1 完整构建

```bash
pnpm build
```

内部执行 `node scripts/build-all.mjs`，构建步骤依次为：

1. **plugins:assets:build** — 构建插件静态资源
2. **tsdown** — TypeScript 编译（输出到 `dist/`）
3. **runtime-postbuild** — 运行时后处理（复制资源、生成覆盖层）
4. **build stamps** — 写入构建时间戳
5. **plugin-sdk DTS** — 生成 Plugin SDK 类型声明

> Windows 下构建内存限制为 4096 MB（`build-all.mjs` 中配置）。

### 4.2 构建用于调试

```bash
# 带 source map 的构建（调试必须）
OUTPUT_SOURCE_MAPS=1 pnpm build
```

`.vscode/tasks.json` 中的 `debug:rebuild` 任务就是执行：

```bash
pnpm clean:dist && pnpm build
```

并自动设置 `OUTPUT_SOURCE_MAPS=1`（通过 tasks.json 的 `env` 字段注入）。当前调试任务还会把
`${HOME}/.nvm/versions/node/v24.11.0/bin` 放到 `PATH` 最前面，避免 VS Code/Cursor 调试器误用
自带的 Node 20。

## 5. 运行项目

### 5.1 补齐 workspace templates

如果启动时报类似 `Missing workspace template: IDENTITY.md` 或 `Missing workspace template: USER.md`，
先从上游仓库下载缺失模板：

```bash
curl -fsSL https://raw.githubusercontent.com/openclaw/openclaw/main/docs/reference/templates/IDENTITY.md \
  -o docs/reference/templates/IDENTITY.md

curl -fsSL https://raw.githubusercontent.com/openclaw/openclaw/main/docs/reference/templates/USER.md \
  -o docs/reference/templates/USER.md
```

下载后可以用 smoke 脚本确认模板完整：

```bash
PATH="${HOME}/.nvm/versions/node/v24.11.0/bin:$PATH" node scripts/lib/workspace-bootstrap-smoke.mjs
```

### 5.2 开发模式（跳过渠道插件，最快启动）

```bash
# 跳过 WhatsApp/Telegram 等渠道，只启动 Gateway 核心
pnpm gateway:dev
```

内部等价于：

```bash
OPENCLAW_SKIP_CHANNELS=1 node scripts/run-node.mjs --dev gateway
```

### 5.3 带文件监听的开发模式

```bash
# 文件变更时自动重建（需要 tmux）
pnpm gateway:watch
```

### 5.4 首次初始化（交互式向导）

```bash
node openclaw.mjs onboard
```

按向导配置 gateway token、工作区、渠道、skills。

### 5.5 直接运行已构建产物

```bash
node openclaw.mjs gateway run
```

## 6. 断点调试

项目已配置好 VS Code 调试，推荐通过 **WSL + VS Code Remote** 调试。

> 当前 `.vscode` 配置固定使用 `${HOME}/.nvm/versions/node/v24.11.0/bin/node`。如果你安装的是
> 其他 Node 22.19+ 或 Node 24 版本，需要同步修改 `.vscode/launch.json` 和 `.vscode/tasks.json`
> 里的 Node 路径。

### 6.1 VS Code 配置（已内置）

**文件：** `.vscode/launch.json`

```jsonc
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Rebuild and Debug Gateway",
      "type": "node",
      "request": "launch",
      "preLaunchTask": "debug:rebuild", // 自动重建再调试
      "runtimeExecutable": "${env:HOME}/.nvm/versions/node/v24.11.0/bin/node",
      "program": "${workspaceFolder}/openclaw.mjs",
      "args": ["gateway", "run"],
      "console": "integratedTerminal",
      "env": {
        "PATH": "${env:HOME}/.nvm/versions/node/v24.11.0/bin:${env:PATH}",
      },
      "sourceMaps": true,
      "smartStep": true,
      "outFiles": ["${workspaceFolder}/dist/**/*.js"],
      "skipFiles": ["<node_internals>/**", "node_modules/**"],
    },
    {
      "name": "Debug Gateway",
      "type": "node",
      "request": "launch",
      "runtimeExecutable": "${env:HOME}/.nvm/versions/node/v24.11.0/bin/node",
      "program": "${workspaceFolder}/openclaw.mjs", // 直接调试，不重建
      "args": ["gateway", "run"],
      "console": "integratedTerminal",
      "env": {
        "PATH": "${env:HOME}/.nvm/versions/node/v24.11.0/bin:${env:PATH}",
      },
      "sourceMaps": true,
      "smartStep": true,
      "outFiles": ["${workspaceFolder}/dist/**/*.js"],
      "skipFiles": ["<node_internals>/**", "node_modules/**"],
    },
  ],
}
```

**`debug:rebuild` Task（`.vscode/tasks.json`）：**

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "debug:rebuild",
      "type": "shell",
      "command": "pnpm clean:dist && pnpm build",
      "options": {
        "env": {
          "OUTPUT_SOURCE_MAPS": "1",
          "PATH": "${env:HOME}/.nvm/versions/node/v24.11.0/bin:${env:PATH}"
        }
      }
    }
  ]
}
```

### 6.2 调试步骤

**方式一：VS Code Remote WSL（推荐）**

1. 在 VS Code 中安装 **WSL 扩展**（`ms-vscode-remote.remote-wsl`）
2. 命令面板 → `WSL: Open Folder in WSL` → 选择 `/mnt/d/repository/github/openclaw`
3. 确认 VS Code 终端中的 Node 版本：`node --version` 应为 `v24.11.0`
4. 如果已有 gateway 在后台运行，先停止：`pnpm openclaw gateway stop`
5. 按 `F5` 选择 `Rebuild and Debug Gateway` 或 `Debug Gateway`
6. 在 TypeScript 源码中打断点，VS Code 通过 source map 自动映射到 `dist/` 中的 JS

**方式二：手动 --inspect**

```bash
# WSL 中手动启动调试
OUTPUT_SOURCE_MAPS=1 pnpm build
pnpm openclaw gateway stop
node --inspect-brk openclaw.mjs gateway run
```

然后在 VS Code 中附加到 `9229` 端口（Add Configuration → Node.js: Attach）。

### 6.3 常见启动失败

**只有 `Process exited with code 1`，没有其它日志**

通常是 VS Code/Cursor 使用了自带的 Node 20。OpenClaw 要求 Node 22.19+，入口会立即退出。当前
`launch.json` 已通过 `runtimeExecutable` 固定到 Node 24，并通过 `console: "integratedTerminal"` 把
完整 stdout/stderr 打到集成终端。

**提示 `gateway already running` 或 `Port 18789 is already in use`**

说明已有 gateway 占用了默认端口。调试前先执行：

```bash
pnpm openclaw gateway stop
```

### 6.4 Source Map 说明

- TypeScript 源在 `src/`，编译产物在 `dist/`
- 必须带 `OUTPUT_SOURCE_MAPS=1` 构建才有 source map
- `tsconfig.json` 中 `"noEmit": true`，实际编译由 **tsdown** 完成
- 调试时 VS Code 会自动通过 `outFiles: ["dist/**/*.js"]` 找到对应 JS 文件
